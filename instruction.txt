I want to create app that can consume this api:

This is the API Request:

```
curl -X POST https://api.toriitranslate.com/api/v2/upload \
-H "Authorization: Bearer YOUR_API_KEY" \
-F "target_lang=en" \
-F "translator=gemini-2.5-flash" \
-F "font=wildwords" \
-F "text_align=auto" \
-F "stroke_disabled=false" \
-F "min_font_size=12" \
-F "file=@path/to/image.png" \
--output response.json --fail --silent --show-error
```

I want this app to be interactive and asking few questions first before executing. This app should read files on current directory where user run the script from.

1. App should ask for api key, then save it to ~/.torii-translate/config. If api key exists in config file, do not ask again.
2. App should ask which translator to use (model), the choice will be:
    - name = Gemini 2.5 Flash Lite (1 credit), value = gemini-2.5-flash
    - name = Deepseek (1 credit), value = deepseek
    - name = Grok 4.1 Fast (1 credit), value = grok-4-fast
    - name = Kimi K2.5 (2 credits), value = kimi-k2
    - name = GPT 5.1 (2+ credits), value = gpt-5
    - name = Gemini 3 Flash (2+ credits), value = gemini-3-flash
3. App should ask whether user want multiple images or single image translate.
4. If user select single image, ask the file image name in current directory (display the current directory name in the question so user can know they are in correct directory).
5. If user select multiple image, ask the start image name end end image name. (the input should be like this: start = 001.png, end = 003.png --> the api should be called 3 times for 001.png, 002.png, 003.png). Files are processed one at a time in order. Start and end filenames must share the same extension and zero-padding width.
6. The api should save the translated result in new directory named translated-result. The output file extension is derived from the API response format, not the input filename.
7. Each image shows an animated spinner with millisecond elapsed time while processing. The completed line shows the total time taken (e.g. 14.5s).
8. In batch mode, pressing Ctrl+C finishes the current file then stops cleanly with a summary.
